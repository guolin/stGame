# 黑客松 Demo 流程

本文档定义大电视演示时的流程。目标是让评委在几分钟内理解项目价值，并看到可玩的 1-4 船互动。

## 1. Demo 总目标

Demo 要让观众记住三件事：

- 这是一个用真实帆船手柄玩的战术模拟器。
- 帆船比赛不仅是往前开，还要读风、读流、选航线。
- 游戏能自动解释规则，让新手理解为什么犯规。

## 2. 推荐总时长

建议控制在 5-7 分钟。

| 阶段 | 时长 | 内容 |
| --- | ---: | --- |
| 项目介绍 | 60-90 秒 | 背景、痛点、方案、硬件 |
| 手柄教学 | 45-60 秒 | 说明舵杆如何控制船 |
| 规则教学 | 60-90 秒 | 演示左舷 / 右舷受风和犯规减速 |
| 4 船比赛 | 120-180 秒 | 4 个学生实际比赛 |
| 赛后总结 | 30-60 秒 | 展示排名、犯规和战术建议 |

## 3. 开场项目介绍

网页打开后，先进入项目介绍页面或演示模式。

需要呈现：

- 项目名称：《帆船战术对抗模拟器》。
- 一句话介绍：用特制蓝牙帆船手柄，把风、水流、航线选择和竞赛规则变成可玩的战术训练。
- 黑客松亮点：
  - 真实手柄。
  - 大屏多人互动。
  - 自动规则解释。
  - 教学和比赛一体化。

如果把 PPT 放进游戏中，建议每一页都能用手柄或键盘切换，并且最后一页直接进入教学。

## 4. 手柄教学

教学只讲一个动作：

```text
拨动舵杆 -> 船向左 / 向右转
```

大屏提示：

- 舵杆居中：直行。
- 舵杆向左：左转。
- 舵杆向右：右转。
- 船速越快，转向越明显。

如果 Demo 当前还没有调帆硬件输入，可以把调帆做成自动模式，避免玩家同时学习太多东西。

## 5. 战术教学

战术教学只讲三个概念：

### 5.1 不能直接顶风

让一条船朝风向冲，显示：

```text
顶风：船速下降
```

### 5.2 风区会影响速度

让一条船进入强风区，显示：

```text
强风区：船速提升
```

### 5.3 水流会推走航迹

让一条船经过水流区，显示：

```text
水流：对地航迹被推偏
```

## 6. 规则教学

规则教学优先演示一左一右两条船交叉。

大屏提示：

```text
规则10：左舷受风船应避让右舷受风船
```

如果发生碰撞：

```text
2号船犯规：违反规则10
处罚：减速 3 秒
```

Demo 版本的处罚选择“减速 3 秒”，因为它比真实转圈处罚更容易被现场观众理解，也更适合学生快速实现。

## 7. 4 船比赛

主比赛建议使用 IO 场地或复杂 4 标场地。

比赛规则：

- 4 个学生各控制一条船。
- 比赛时间建议 2-3 分钟。
- 屏幕持续显示风向、水流、风区、航迹和当前排名。
- 犯规时暂停或慢放 1 秒，并显示规则解释。
- 犯规船强制减速 3 秒。

为了保证 Demo 稳定，建议准备两个按钮：

- `重新开始比赛`。
- `触发规则演示场景`。

## 8. 赛后总结

比赛结束后显示：

- 排名。
- 每条船用时。
- 犯规次数。
- 被减速次数。
- 最快速度。
- 关键战术提示。

赛后提示最多三条：

- “1号船选择了右侧强风区，因此上风段更快。”
- “2号船违反规则10，被减速 3 秒。”
- “3号船过早进入 Layline，多走了一段距离。”

## 9. 现场兜底方案

必须准备：

- 手柄无法连接时，切到键盘或自动演示模式。
- 电视自带安卓浏览器性能不足时，切到低性能模式或外接浏览器设备。
- 4 船太乱时，切到 2 船规则演示。
- 风区和水流看不清时，打开高亮图层。
- 规则不稳定时，使用预设规则情景场地。
- 时间不足时，跳过完整比赛，只展示“手柄 + 犯规解释”。

Demo 的最高优先级是稳定讲清楚项目价值。
