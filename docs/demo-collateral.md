# Demo 副产品清单

黑客松 Demo 不只需要网页游戏本身，还需要一套能帮助观众快速理解项目的配套材料。

## 1. 副产品范围

必须准备：

- 海报。
- 传单。
- 项目网站。

可选准备：

- 二维码入口。
- 一页式项目说明 PDF。
- 演示视频。
- 评委问答卡片。

## 2. 海报

用途：

- 放在展位或大电视旁边。
- 让路过的人在 5 秒内知道项目是什么。

内容建议：

- 项目名称：《帆船战术对抗模拟器》。
- 一句话介绍：用蓝牙帆船模型手柄学习风、水流、航线和竞赛规则。
- 三个亮点：
  - 大屏 1-4 人对抗。
  - 风区、水流和航线选择可视化。
  - 自动解释帆船比赛规则。
- 一张主视觉：大电视上的 4 船比赛画面或模拟截图。
- 二维码：打开项目网站或在线 Demo。

设计要求：

- 远距离可读。
- 不堆规则文字。
- 强调硬件手柄和大屏互动。

## 3. 传单

用途：

- 给评委或观众带走。
- 简短解释项目背景和技术方案。

内容建议：

- 项目是什么。
- 为什么帆船比赛难学。
- 我们如何用游戏和手柄解决。
- Demo 怎么玩。
- 使用了哪些技术。
- 学生在项目中负责什么。
- 二维码和联系方式。

传单建议做成一页双面或一页单面，不要超过一张 A4。

## 4. 项目网站

项目网站可以和游戏同域，也可以是单独页面。它的作用是让评委在 Demo 后继续了解项目。

必须包含：

- 项目介绍。
- 在线 Demo 入口。
- 核心功能截图。
- 硬件手柄说明。
- 团队介绍。
- 技术架构。
- 后续计划。

建议结构：

```text
首页
  项目一句话介绍
  打开 Demo
  看讲解

功能
  船与风
  水流和风区
  规则裁判
  4 人大屏对抗

硬件
  蓝牙帆船模型手柄
  1-4 通道

团队
  负责人
  A 组
  B 组

下载 / 联系
  海报
  传单
  项目仓库或二维码
```

## 5. 与游戏页面的关系

网站、项目介绍页和 PPT 不应完全割裂：

- 网站用于外部访问和赛后传播。
- 游戏内项目介绍页用于现场开场。
- 海报和传单用于展位传播。

三者应该使用同一套项目名称、主视觉、颜色、口号和二维码。

## 6. 最低交付标准

如果时间紧，最低需要：

- 一张海报。
- 一张 A4 传单。
- 一个网站首页。
- 一个二维码，能打开项目介绍或 Demo。

如果只能优先做一个，先做网站首页，因为它可以同时承载二维码、项目介绍、截图和 Demo 入口。
