# 竞赛规则与自动裁判设计

本文档定义 MVP 自动裁判范围。目标不是完整复刻抗议委员会流程，而是在最常见会船、绕标和处罚场景下给出稳定、可解释、适合大电视 Demo 的裁决。

## 1. MVP 规则范围

黑客松 Demo 必须优先稳定实现：

- 规则 10：不同舷受风时，左舷受风船避让右舷受风船。
- 碰撞检测。
- 犯规船强制减速 3 秒。
- 大屏规则提示：谁犯规、违反哪条规则、为什么。

训练版本继续扩展：

- 规则 10：不同舷受风时，左舷受风船避让右舷受风船。
- 规则 11：同舷受风且重叠时，上风船避让下风船。
- 规则 12：同舷受风且不重叠时，后船避让前船。
- 规则 13：正在迎风换舷的船避让其他船。
- 规则 14：在合理可能的情况下避免接触。
- 规则 15：刚取得航权时，需给对方初始避让空间。
- 规则 16：航权船改变航向时，需给对方继续避让空间。
- 规则 18：标位空间。
- 航标接触规则。
- 起航抢航 OCS。
- 一圈和两圈解脱处罚。

## 2. 基础避让规则

| 情况 | 游戏判定 |
| --- | --- |
| 两船不同舷受风 | 左舷受风船避让右舷受风船 |
| 两船同舷受风且重叠 | 上风船避让下风船 |
| 两船同舷受风且不重叠 | 后船避让前船 |
| 一条船正在迎风换舷 | 正在换舷的船避让其他船 |

界面必须明确显示：

- 左舷受风 / Port Tack。
- 右舷受风 / Starboard Tack。

避免玩家把“左舷 / 右舷受风”误解为船正在向左或向右行驶。

## 3. 航权船限制

航权船不能无限制逼迫避让船：

- 刚取得航权时，必须先给对方避让空间。
- 改变航向时，必须给对方继续避让的空间。
- 即使有航权，在合理可能的情况下仍然需要避免碰撞。

因此自动裁判不能采用“发生碰撞后永远处罚无航权船”的简单逻辑。

## 4. 重叠判定

系统应按照船尾横向线判断前后关系：

- 一条船完全位于另一条船船尾横线之后，属于明显在后。
- 双方都不明显在另一条船之后，视为重叠。
- 同舷受风时始终使用该判定。
- 在特定绕标或顺风情况下，不同舷受风也可能需要判断重叠。

教学模式中可短暂显示“重叠线”，帮助玩家理解裁决。

## 5. 标位规则

航标周围建立半径为三倍船长的标位区。

当第一条船进入标位区时：

- 若两船重叠，外侧船应给内侧船标位空间。
- 若两船没有重叠，后进入标位区的船应给先进入的船标位空间。
- 标位空间应足够让有权船驶向、绕过并离开航标。

教学模式显示标位区圆圈。竞赛模式默认隐藏，只在发生规则风险时淡入。

## 6. 航标接触

比赛中不得接触当前航段的航标。

判定要求：

- 使用船体碰撞区域，而不是船图标中心点。
- 只对当前航段相关航标触发处罚。
- 触碰无关航标可作为碰撞反馈，但不应自动产生竞赛处罚。

## 7. 起航抢航

起航信号发出时，船体必须完全位于赛前侧，之后才能合法向航线侧穿越起航线。

提前越线处理：

1. 显示“OCS / 抢航”。
2. 玩家必须返回起航线赛前侧。
3. 船体完全返回后，重新穿越起航线。
4. 未修正便完成航线，判定未完成或取消成绩。

## 8. 自动裁判优先级

每次规则判定按以下顺序执行：

1. 船是否正在执行处罚或返回起航。
2. 船是否正在迎风换舷。
3. 是否适用标位规则。
4. 当前基础航权关系。
5. 航权是否刚刚发生变化。
6. 航权船是否改变航向。
7. 双方是否采取了合理避碰行动。
8. 是否发生实际接触。

系统应保存碰撞前约 1-2 秒的状态快照，避免只根据碰撞瞬间船位判断责任。

## 9. Demo 处罚机制

黑客松版本采用强制减速处罚，而不是要求玩家现场完成真实转圈解脱。

流程：

1. 系统检测碰撞或规则违规。
2. 屏幕显示规则提示。
3. 犯规船立即进入 3 秒减速状态。
4. 减速状态结束后恢复正常比赛。

减速效果要求：

- 船速明显下降，观众能看出来。
- 船体或尾迹出现明显处罚视觉状态。
- 玩家 HUD 显示倒计时，例如 `Penalty 2.1s`。
- 同一事件只触发一次处罚，避免一直接触导致重复处罚。

Demo 文案示例：

```text
2号船犯规：违反规则10
左舷受风船没有避让右舷受风船
处罚：减速 3 秒
```

## 10. 训练版处罚机制

### 10.1 真实规则模式

- 违反会船规则通常可以执行两圈解脱。
- 碰标可以执行一圈解脱。
- 每一圈应包含一次迎风换舷和一次顺风换舷。

游戏流程：

1. 发生犯规后出现处罚标志。
2. 玩家应尽快驶离其他船。
3. 在规定时间内开始执行处罚。
4. 系统自动识别完整转圈。
5. 完成后清除处罚。

### 10.2 初学者模式

初学者模式保留转圈处罚，但提供：

- 建议转向箭头。
- 已完成角度进度环。
- “尚缺一次换舷 / 顺风换舷”的提示。
- 自动慢放。

不建议仅使用“减速三秒”代替转圈，因为这会削弱帆船规则训练价值。

### 10.3 未执行处罚

- 规定时间内未开始：增加时间处罚并持续警告。
- 未完成处罚便冲线：不记录有效冲线。
- 严重碰撞或通过犯规取得明显优势：可判退出比赛。

## 11. 规则引擎状态

每条船保存：

```text
position
heading
velocityThroughWater
velocityOverGround
tack
windwardSide
tackingState
overlapState
rightOfWayState
markZoneState
penaltyState
slowdownPenaltyUntil
courseLeg
```

规则引擎不直接依赖 UI，通过规则事件输出：

```text
RULE_WARNING
RULE_BREACH
PENALTY_ASSIGNED
SLOWDOWN_STARTED
SLOWDOWN_COMPLETED
PENALTY_STARTED
PENALTY_COMPLETED
EXONERATED
OCS
MARK_CONTACT
COURSE_ERROR
```

这种结构便于未来增加 AI、回放和在线同步。

## 12. 可解释事件文案

轻度风险：

```text
玩家2应避让：左舷受风船正在接近右舷受风船。
```

基础违规：

```text
玩家2违反规则10：左舷受风船未避让右舷受风船。
处罚：减速 3 秒。
```

航权船限制：

```text
玩家1原本拥有航权，但改变航向时没有给玩家2足够避让空间。
玩家1违反规则16.1。
```

标位：

```text
玩家2进入三船长标位区时在外侧，应给玩家1标位空间。
玩家2未给足空间，违反规则18。
```
