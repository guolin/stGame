# 项目背景

## 1. 项目定位

《帆船战术对抗模拟器》是一个专业但适合演示的帆船竞赛训练项目。它把真实帆船比赛中的三个核心问题可视化：

- 风：风向会左右摆动，风速会变化，局部风区会影响船速和航线选择。
- 流：水流会把船推离原本航线，影响起航、绕标和冲线。
- 规则：船与船相遇时，不只是比谁快，还要判断左/右舷受风、上风/下风、前后船、换舷和避碰义务。

这个项目不是为了做一个大型商业游戏，而是为了黑客松和课堂演示：让观众、评委、学生通过一场短比赛理解帆船战术和规则。

## 2. Demo 场景

主要演示场景是大电视，其次是 iPad。

推荐 Demo 画面：

- 大电视显示完整赛场、风区、水流、航迹、规则提示和项目介绍。
- 4 个学生站在电视前，每人拿一个特制蓝牙帆船模型手柄。
- 每个手柄控制一条船，最多 4 条船同时比赛。
- 犯规时，屏幕直接显示“谁犯规、违反哪条规则、为什么”，犯规船强制减速 3 秒。
- 比赛结束后，屏幕显示排名、关键犯规和战术提示。

## 3. 项目目标

黑客松版本的目标是做出一个稳定、好讲、能互动的基础脚手架：

- 打开网页后可以先展示项目介绍。
- 介绍结束后进入教学环节。
- 教学结束后进入 1-4 人比赛。
- 比赛中能展示风向、风速、水流和风区。
- 比赛中能检测几类核心规则并给出提示。
- 犯规后使用强制减速 3 秒作为 Demo 处罚。
- 整体体验活泼、清晰、适合大屏观看。

## 4. 关键设计原则

### 4.1 Demo 优先

多数开发时间应投入在演示效果上：

- 观众能快速看懂当前发生了什么。
- 评委能快速理解项目创新点。
- 手柄操作稳定，不需要现场解释太多技术细节。
- 犯规提示要醒目、短、可复述。

### 4.2 专业感保留

即使是黑客松 Demo，也要保留帆船比赛的专业判断：

- 不使用“汽车开船”的操控逻辑。
- 船速必须受风角影响。
- 风区、水流、Layline 和禁航角要对战术有实际影响。
- 规则至少覆盖 RRS Part 2 中最常见的 10-16 类判断。

### 4.3 学生可扩展

当前阶段要先把基础框架搭好，让学生后续可以在清晰边界内完善：

- 帆船比赛组：调船速、调舵效、调风、水流、赛道和手感。
- 规则组：实现航权判断、规则提示、犯规减速和视觉反馈。
- 演示内容组：整理 PPT 内容、教学流程、Demo 脚本和大屏视觉包装。

如果只有 4 个学生，建议分成两个两人小组，但每组必须承担一个 Demo 展示交付物，而不是只做底层功能。

## 5. 黑客松版本的取舍

暂时不追求：

- 完整真实抗议流程。
- 两圈或一圈真实解脱处罚。
- 高精度船型 VPP。
- 在线联机。
- 复杂多船队赛规则。

优先追求：

- 1-4 船能稳定跑。
- 蓝牙手柄输入能映射到对应船。
- 3 种左右可演示场地足够清晰。
- 风、风摆、阵风和风区能看见且影响结果；水流作为可选扩展。
- 规则提示能帮助观众学习。
- Demo 流程能从项目介绍自然切到教学和比赛。
