# 《帆船战术对抗》Web 游戏 PRD

版本：V0.1

产品形态：2D 俯视角、1-4 人同屏、蓝牙帆船模型手柄操控、帆船竞赛操控与规则训练模拟器

规则基准：World Sailing《2025-2028 帆船竞赛规则》，该版本自 2025 年 1 月 1 日起生效。

## 1. 产品定义

《帆船战术对抗》不是单纯的“开船游戏”，而是一个面向黑客松、大电视展示和帆船教学的专业战术对抗模拟器。它把帆船竞赛中看不见的判断过程可视化：

- 操控：船头朝向、转舵、调帆、换舷。
- 环境：风向、风速、阵风、弱风区、水流。
- 战术：起航位置、换舷时机、Layline、抢占有利航线。
- 规则：左/右舷受风、上风/下风、前后船、标位规则和解脱处罚。

产品应同时满足两个价值：

- 对普通玩家，它是一局 3-8 分钟的 1-4 人对抗游戏。
- 对帆船学习者，它是一套能够解释“为什么输、哪里犯规、怎样更快”的训练工具。
- 对黑客松评委，它是一个“硬件手柄 + 大屏互动 + 规则解释”的完整 Demo。

## 1.1 黑客松 Demo 定位

Demo 主场景是大电视，4 个学生各拿一个特制蓝牙帆船模型手柄进行比赛。每个手柄通道控制一条船，最多 4 条船。

Demo 流程：

```text
项目介绍 -> 手柄教学 -> 战术/规则教学 -> 1-4 船比赛 -> 犯规提示 -> 赛后总结
```

黑客松版本优先保证：

- 现场能稳定演示。
- 大屏远距离可读。
- 手柄输入能被观众理解。
- 风、风摆、阵风、风区和规则提示有明显视觉反馈；水流作为可选扩展。
- 犯规后使用强制减速 3 秒作为清晰、易懂的 Demo 处罚。

## 2. 产品目标

### 2.1 核心学习目标

玩家在完成一局游戏后，应当能够理解：

- 船不能直接顶风航行。
- 同一条船在不同风角下速度不同。
- 调帆不正确会显著减速。
- 水流会改变对地航迹，但不会直接改变相对水速。
- 提前或过晚上 Layline 都可能损失距离。
- 有航权不等于可以故意碰撞。
- 换舷、改变航向和进入标位区会改变双方权利义务。

### 2.2 MVP 成功标准

- 新玩家在 3 分钟教学后能够独立完成比赛。
- 1-4 名玩家可以通过蓝牙帆船模型手柄完成对局。
- 键盘输入作为开发和现场兜底方案。
- 系统能够自动判断主要避让规则和标位规则。
- 每次判罚都能显示“谁犯规、违反什么规则、为什么”。
- 比赛结束后能够回放关键事件，而不只是显示胜负。

### 2.3 MVP 暂不纳入

- 精细帆型选择和帆具损伤。
- 波浪、涌浪、横摇和翻船。
- 多船队赛。
- 完整抗议委员会流程。
- 不同船型的高精度 VPP。
- 大规模在线联机和排位系统。

## 3. 目标用户

| 用户 | 主要需求 |
| --- | --- |
| 帆船初学者 | 学会迎风航行、换舷、调帆和基本规则 |
| 青少年帆船学员 | 通过游戏理解教练讲解的抽象概念 |
| 俱乐部及教练 | 用于课堂演示、规则情景训练和复盘 |
| 有竞赛经验的玩家 | 练习起航、Layline、覆盖和规则博弈 |
| 普通多人游戏玩家 | 快速、直观且有对抗性的本地竞技游戏 |
| 黑客松评委和观众 | 快速理解硬件交互、帆船战术和自动规则解释 |

## 4. 核心玩法

### 4.1 标准比赛流程

1. 打开网页，进入项目介绍或直接进入比赛设置。
2. 选择船只数量：1、2、3 或 4 条船。
3. 选择赛道、风况、水流、规则难度和控制方式。
4. 映射蓝牙手柄通道 1-4 到船只 1-4。
5. 进入起航倒计时。
6. 玩家在起航线后争夺位置。
7. 起航信号发出后合法穿越起航线。
8. 按指定顺序绕行各个航标。
9. 穿过终点线。
10. 系统处理犯规减速、航线错误和未完成条件。
11. 显示比赛结果、轨迹、规则事件和战术分析。

### 4.2 MVP 场地

Demo 阶段固定 4 个场地，不追求数量多，追求稳定好讲。

#### 简单场地

```text
起航线 -> 1 标 -> 终点线
```

用于手柄教学和新手第一次试玩。

#### IO 场地

```text
起航线 -> 1 标 -> 2 标 -> 终点线
```

用于主 Demo，复杂度适中，适合 2-4 船比赛。

#### 复杂 4 标场地

```text
起航线 -> 1 标 -> 2 标 -> 3 标 -> 4 标 -> 终点线
```

用于 4 船完整对抗，展示风区、水流和多次会船。

#### 规则情景场地

```text
预设船位 -> 触发会船 / 换舷 / 绕标规则 -> 显示裁决
```

用于教学和兜底 Demo，确保规则提示能稳定出现。

### 4.3 后续标准航线

后续训练版本可加入简化迎尾风航线：

```text
起航线 -> 上风标 -> 下风门 -> 上风标 -> 终点线
```

### 4.4 获胜条件

第一条满足以下全部条件并合法冲过终点的船获胜：

- 已合法起航。
- 按正确顺序完成所有航段。
- 在规定一侧绕过航标。
- 没有未执行的处罚。
- 未被判退出或取消资格。

正式规则中，船需要在起航信号后从起航线的赛前侧穿越至航线侧，并按正确顺序完成航标和终点。

## 5. 操控方案

### 5.1 蓝牙帆船模型手柄

正式 Demo 使用特制蓝牙手柄。手柄外观是一个帆船模型，1、2、3、4 通道分别对应 1、2、3、4 条船。

每个通道至少输出舵杆值：

```text
rudder = -100 ~ 100
```

映射方式：

| 输入值 | 含义 |
| --- | --- |
| -100 | 最大左舵 |
| 0 | 舵居中 |
| 100 | 最大右舵 |

内部归一化：

```text
turn = rudder / 100
```

蓝牙硬件每个连接器只有一个轴，用于控制船舵。黑客松 Demo 不做玩家手动调帆，系统自动调帆，玩家只控制方向。

### 5.2 备用输入

| 玩家 | 输入 | 功能 |
| --- | --- | --- |
| 玩家 1 | A / D 或左 / 右方向键 | 向左、向右转舵 |
| 玩家 1 | W / S 或上 / 下方向键 | 调试参数或预留 |
| 玩家 2 | 可配置按键 | 向左、向右转舵 |
| 公共 | Space | 暂停 / 继续 |
| 公共 | R | 重新开始 |

键盘控制用于开发、调试和现场兜底。正式大电视演示优先使用蓝牙帆船模型手柄。

### 5.3 模拟控制

开发阶段需要支持“模拟控制”：

- 没有蓝牙手柄时，可以用键盘左右键或 `A/D` 模拟舵杆。
- 可以用自动演示输入让船按预设路线移动。
- 讲解模式和项目介绍页可以用脚本控制船，而不是依赖真实玩家操作。

模拟控制的目标是保证没有硬件时也能开发、测试和演示。

### 5.4 操控原则

不建议使用“按键直接改变船头角度”的方式，而应模拟基本舵效：

- 按住转向键产生舵角。
- 船速越高，舵效越明显。
- 船速过低时很难迅速转向。
- 舵角过大会增加阻力。
- 松开按键后舵逐渐回中。
- 换舷过程中产生明显速度损失。

这样能够避免游戏变成没有惯性的“坦克转向”。

## 6. 帆船运动模型

### 6.1 风场

游戏世界应包含两层风数据：

- 全局风：基准风向、基准风速、当前主要受风舷。
- 局部风：阵风区、弱风区、风向左摆或右摆、风速渐变、随时间移动的风带。

风箭头越长、越亮，代表风速越大。

黑客松 Demo 必须明确展示：

- 风向会左右摆动。
- 局部风区会改变风速或风向。
- 船进入风区后速度或航向选择会受到影响。

风摆必须区分两类：

- 持续风摆：风向持续向左或向右变化一段时间。
- 钟摆式风摆：风向围绕基准方向周期性左右摆动。

阵风是小面积局部区域，可以移动，可以让风速变大、变小，也可以让风向发生偏移。

### 6.2 水流

水流作为额外速度矢量加入船的对地运动：

```text
对地速度 = 相对水速度 + 水流速度
```

水流会影响：

- 实际航迹。
- 到达 Layline 的时间。
- 起航线接近速度。
- 绕标时的横向漂移。

Demo 中水流可以固定在一个或多个区域，重点是让观众看见“船被推偏”。

界面应区分：

- STW：相对水速度。
- SOG：对地速度。

MVP 默认可只显示一个简化速度值，高级模式再同时显示两者。

### 6.3 船速模型

目标船速由以下参数决定：

```text
目标船速 =
船型极曲线速度
* 调帆效率
* 局部风速系数
* 操舵阻力系数
* 换舷状态系数
```

主要变量：

- 真风角 TWA。
- 真风速 TWS。
- 船型 Polar Data。
- 当前帆角。
- 理想帆角。
- 舵角。
- 船体转动惯性。
- 当前是否处于换舷或顺风换舷状态。

每种船型应有独立 Polar Data。Polar Data 描述某条船在不同真风速和真风角下能达到的相对水速度。水流不改变 Polar Data，只在最后影响对地速度。

```text
targetSTW = polarSpeed(boatType, TWS, TWA)
SOG = actualSTW vector + current vector
```

黑客松阶段默认只支持 OP 和 Topper 二选一。数据结构保留未来加入其他船型的能力。

### 6.4 禁航角

船头过于接近来风方向时进入禁航角：

- 帆开始抖动。
- 船速迅速下降。
- 舵效逐渐减弱。
- 界面提示“顶风”或“帆失速”。

建议初始禁航角设置为来风方向两侧各 40-45 度，具体数值由船型参数控制。

### 6.5 调帆

调帆应直接影响速度：

- 帆收得过紧：横向力和阻力增加。
- 帆放得过松：帆无法有效受风。
- 接近理想角度：获得最高效率。
- 接近失速边界时显示帆抖动动画。

初级模式可以提供“理想帆角提示线”，高级模式关闭提示。

## 7. 游戏模式

### 7.1 教学模式

按顺序解锁：

1. 转向和船体惯性。
2. 收帆、放帆与理想帆角。
3. 顶风、抢风航行和迎风换舷。
4. 顺风航行和顺风换舷。
5. 起航倒计时。
6. Polar Data：不同风角下速度不同。
7. 持续风摆与钟摆式风摆。
8. 移动阵风与路线选择。
9. 左右舷受风规则。
10. 上风船与下风船。
11. 标位区与绕标。
12. 完整比赛。

每个教学关卡只教授一个概念。

### 7.2 双人本地对抗

保留双人模式，主要用于规则教学和简化 Demo。

- 同一键盘。
- 键盘加手柄。
- 双手柄。
- 支持暂停和重开。
- 支持快速再来一局。

### 7.3 1-4 人本地对抗

黑客松主模式。

- 1 条船：教学和计时。
- 2 条船：规则演示。
- 3 条船：中等复杂度对抗。
- 4 条船：大电视主 Demo。

每条船绑定一个手柄通道。

### 7.4 项目介绍模式

网页打开后可以进入项目介绍模式，用于替代或承载 PPT：

- 展示项目名称、背景和痛点。
- 展示特制蓝牙帆船模型手柄。
- 展示风、风摆、阵风、风区、可选水流和规则解释能力。
- 最后一页直接进入教学或比赛。

### 7.5 单人计时赛

玩家与自己的最佳路线竞争，训练：

- 换舷次数。
- Layline 判断。
- 风区选择。
- 调帆效率。
- 绕标路线。

### 7.6 规则情景训练

系统直接加载典型局面：

- 左舷船与右舷船交叉。
- 下风船迎风逼迫上风船。
- 两条船接近上风标。
- 标位区内形成或解除重叠。
- 航权船突然改变航向。
- 两船同时换舷。

玩家操作后立即显示裁决和解释。

### 7.7 后续模式

- AI 对手。
- 在线双人。
- 教练创建自定义局面。
- 多船船队赛。
- 历史比赛局面复现。

## 8. 界面设计

### 8.1 当前界面区域

大屏优先级高于桌面网页。所有关键状态必须在电视上远距离可读。

左上风况面板显示：

- 风向箭头。
- 基准风速。
- 当前船或选中船的受风舷。

建议增加：

- 风向角度。
- 当前局部风速。
- 风摆趋势。

右上水流面板显示：

- 水流方向。
- 水流速度。

顶部中央根据阶段显示：

| 阶段 | 显示 |
| --- | --- |
| 起航前 | 倒计时 |
| 比赛中 | 已用时间 |
| 处罚中 | 减速倒计时 |
| 完赛后 | 冲线时间 |

玩家状态面板建议包含：

- 玩家编号和颜色。
- 船速。
- 航行状态。
- 左/右舷受风。
- TWA。
- 调帆效率。
- 当前名次。
- 未完成处罚。
- 当前航段。
- 手柄通道编号。
- 犯规减速倒计时。

底部控制和图层建议包含：

- 暂停 / 继续。
- 慢放。
- 重开。
- 风。
- 流。
- 航迹。
- Layline。
- 禁航角。
- 标位区。
- 规则关系。
- 理想帆角。
- 碰撞预测。
- 船速极曲线提示。

### 8.2 船只视觉反馈

每条船至少需要表达：

- 船头方向。
- 船帆所在一侧。
- 玩家颜色。
- 航迹。
- 换舷状态。
- 处罚状态。
- 当前是否有航权。

不建议长期用红色表示玩家 1，因为红色还需要用于犯规和危险提示。可以保留玩家颜色，但规则警告应采用独立图标和描边。

### 8.3 规则提示

轻度风险示例：

```text
玩家2应避让：左舷受风船正在接近右舷受风船。
```

已发生违规示例：

```text
玩家2违反规则10：左舷受风船未避让右舷受风船。
处罚：减速 3 秒。
```

复杂判定示例：

```text
玩家1原本拥有航权，但改变航向时没有给玩家2足够避让空间。
玩家1违反规则16.1。
```

提示应尽量使用自然语言，规则编号作为辅助信息。

## 9. 犯规处罚与学习反馈

黑客松 Demo 版本采用强制减速处罚：

- 犯规船强制减速 3 秒。
- 屏幕显示规则编号、犯规船、被影响船和自然语言解释。
- 减速期间船体出现明显视觉状态，例如灰色尾迹、减速图标或倒计时。

真实训练版本后续再加入一圈、两圈解脱处罚。

## 10. 比赛结束与复盘

### 10.1 结果页面

显示：

- 最终名次。
- 总时间。
- 处罚次数。
- 抢航次数。
- 碰标次数。
- 碰撞次数。
- 被强制减速次数。
- 换舷次数。
- 平均速度和最高速度。
- 实际航程。
- 理论最短航程。
- 调帆效率。
- 在强风区停留时间。

### 10.2 战术评价

系统生成不超过三条高价值建议，例如：

- “你在距离上风标过远时进入 Layline，额外航行了 86 米。”
- “第 2 次换舷后帆收得过紧，持续损失约 7 秒。”
- “你拥有右舷受风航权，但改变航向过快，没有给对手避让空间。”
- “右侧水流将船推向航线外侧，本局左侧起航更有利。”

### 10.3 回放功能

回放需要支持：

- 暂停。
- 0.25x、0.5x、1x、2x。
- 前后跳转关键事件。
- 显示双方输入。
- 显示风流场。
- 显示航权变化。
- 显示碰撞前预测轨迹。
- 查看裁判判定依据。

## 12. 非功能需求

### 12.1 性能

- 桌面浏览器稳定运行在 60 FPS。
- 操作输入到画面反馈低于 50 ms。
- 物理模拟采用固定时间步长。
- 慢放和回放时不能改变物理判定结果。
- 同一初始条件和输入序列必须得到相同结果。

### 12.2 浏览器

MVP 支持：

- Chrome。
- Edge。
- Safari。
- Firefox。

优先适配大电视和 iPad。推荐最低分辨率为 1280 x 720，大电视演示建议适配 1920 x 1080。

黑客松 Demo 需要尽量能在电视机自带安卓浏览器上运行。因此技术选择要保守：

- 避免依赖过新的浏览器 API。
- 避免必须开启高性能 WebGPU。
- 优先使用 Canvas / WebGL 的基础能力。
- 保持资源体积较小，减少大图片和复杂后处理。
- 首页、项目介绍和讲解页必须在低性能浏览器上流畅打开。
- 如果安卓电视浏览器性能不足，应准备 Chrome/Edge 外接设备作为兜底。

### 12.3 可用性

- 色彩不是区分玩家和规则状态的唯一方式。
- 船只增加编号、轮廓和形状标识。
- 所有关键操作允许重新绑定。
- 教学文本支持中英文切换。

## 13. 验收标准

MVP 至少满足以下测试：

1. 船进入禁航角后能够明显减速。
2. 正确调帆比错误调帆获得更高船速。
3. 相同风况下，水流能够改变对地航迹。
4. 1-4 个手柄通道能够映射到 1-4 条船。
5. 起航前越线不会被判合法起航。
6. 左舷受风船撞击右舷受风船时，系统能够正确判断基本避让义务。
7. 犯规船能够被强制减速 3 秒。
8. 大屏能显示犯规船、规则编号和自然语言解释。
9. 同舷重叠时能够正确识别上风船与下风船。
10. 换舷中的船能够被识别为避让船。
11. 航权船突然转向、不给对方避让空间时，系统不会错误处罚对方。
12. 简单场地、IO 场地和复杂 4 标场地可以切换。
13. 项目介绍、教学和比赛能形成完整 Demo 流程。
14. 暂停和慢放不会改变最终比赛结果。
