# UI 与交互框架

本文档定义《帆船战术对抗模拟器》的页面结构、入口路径、比赛设置、HUD 仪表盘和大电视 Demo 交互。目标是让项目不是“直接进入游戏画面”，而是形成一个可以讲解、教学、设置、比赛和复盘的完整体验。

## 1. 总体信息架构

网页打开后进入一个大屏友好的主入口，所有流程从这里开始。

用户提出的三个一级入口是合理的，建议直接作为主菜单的核心结构：

1. 比赛界面：选择人数、难度、场地，然后开始对抗。
2. 讲解模式：讲清楚船为什么这样走，以及比赛规则如何判。
3. 项目介绍：用 PPT + 交互式小演示介绍为什么做这个项目。

```text
首页 / 主入口
  -> 比赛
  -> 讲解模式
  -> 项目介绍
  -> 快速 Demo
  -> 赛后复盘
```

黑客松现场默认路径：

```text
首页
-> 项目介绍
-> 讲解模式
-> 比赛
-> 选择人数 / 难度 / 场地
-> 开始 2 船或 4 船比赛
-> 赛后总结
```

日常调试默认路径：

```text
首页
  -> 比赛
  -> 使用上次配置进入赛场
```

## 2. 首页 / 主入口

首页是大电视上的第一屏，必须让观众立刻知道项目是什么。

### 2.1 首页内容

主视觉：

- 项目名称：《帆船战术对抗模拟器》。
- 副标题：用蓝牙帆船模型手柄学习风、水流、航线和竞赛规则。
- 背景：动态赛场预览，能看到风箭头、水流箭头、4 条船和航迹。

主要入口：

| 入口 | 用途 |
| --- | --- |
| 比赛 | 进入对抗模式，选择人数、难度、场地后开赛 |
| 讲解模式 | 讲解船只运行逻辑、比赛规则和两船相遇 Demo |
| 项目介绍 | 展示项目背景、PPT 和交互式项目亮点 |
| 快速 Demo | 现场兜底，一键进入预设演示流程 |

### 2.2 首页操作

支持三种操作：

- 鼠标 / 触屏点击。
- 键盘方向键和确认键。
- 蓝牙手柄通道 1 的确认 / 切换按钮，如果硬件支持。

如果硬件暂时没有按钮，则首页可以保留触屏或鼠标作为现场操作方式。

在安卓电视浏览器上，可能没有稳定鼠标或触屏输入，因此首页还应支持最基础的键盘 / 遥控器方向键操作。

## 3. 比赛界面

比赛界面是“真的开始玩”的入口。它不承担项目介绍任务，只负责把比赛配置清楚、快速地选完。

### 3.1 比赛配置流程

```text
比赛入口
-> 选择人数
-> 选择难度 / 天气
-> 选择场地
-> 手柄检查
-> 开始比赛
```

为了适合大电视，配置页面应该少层级、少文字，选项用大卡片。

### 3.2 人数选择

支持 1、2、3、4 人。

推荐默认值：

- Demo 规则演示：2 人。
- 黑客松主比赛：4 人。
- 调试或教学：1 人。

界面显示：

```text
1 人    2 人    3 人    4 人
```

选择人数后，底部显示启用的船位和手柄通道：

```text
1号船 Ch1
2号船 Ch2
3号船 Ch3
4号船 Ch4
```

### 3.3 难度 / 天气选择

难度可以做成两层，但 Demo 阶段建议合并成一个选择：

| 选项 | 含义 | 适合场景 |
| --- | --- | --- |
| 1 入门 / 小风天 | 船速慢，提示多，规则少 | 第一次玩 |
| 2 标准 / 正常风 | 船速适中，风区和水流明显 | 主 Demo |
| 3 竞赛 / 大风天 | 船速快，规则更严格 | 高级演示 |

如果 UI 上要同时显示“难度”和“天气”，建议这样表达：

```text
难度 1：小风天，辅助多
难度 2：正常风，标准比赛
难度 3：大风天，速度快，犯规更危险
```

这样学生不会同时维护两套复杂配置。

### 3.4 场地选择

场地用 ABCD 或具名场地都可以，但需要和展示话术一致。

建议映射：

| 场地 | 展示名 | 用途 |
| --- | --- | --- |
| A | 简单迎风场地 | 1 标，教学和快速试玩 |
| B | IO 场地 | 1 标、2 标、冲终点，主 Demo |
| C | 三角标场地 | 三角路线，展示转向和绕标 |
| D | 复杂 4 标场地 | 多标对抗，适合 4 人 |

场地卡片必须有小地图预览：

- 起航线。
- 航标编号。
- 终点线。
- 风向箭头。
- 水流区。
- 风区。

### 3.5 开始比赛

点击“开始比赛”后进入准备倒计时：

```text
3
2
1
起航
```

比赛开始前必须完成：

- 根据人数启用对应船只。
- 根据难度加载风、水流、规则严格度。
- 根据场地加载航标和终点。
- 检查手柄通道。

## 4. 讲解模式

讲解模式不是比赛设置，也不是 PPT。它是用互动小 Demo 讲清楚帆船运行和竞赛规则的模式。

讲解模式分成三块：

```text
讲解模式
  -> 船只运行逻辑
  -> 比赛规则
  -> 两船相遇 Demo
```

### 4.1 船只运行逻辑

目标：讲清楚船为什么能走、为什么不能顶风、为什么角度不同速度不同。

小节建议：

1. 风从哪里来。
2. 帆角如何影响船速。
3. 船头和风的夹角不同，速度不同。
4. 不能正顶风。
5. 顺风、侧风、迎风航行的差别。
6. 水流只改变对地航迹。

交互方式：

- 左侧显示一条船。
- 右侧显示风向箭头和速度曲线。
- 用户拖动船头角度或舵杆，船速仪表实时变化。
- 用户打开 / 关闭水流，航迹立刻变化。

这里必须调用和比赛相同的船体、风、水流、速度计算组件。

### 4.2 比赛规则

目标：讲清楚最重要的规则，而不是背完整规则书。

小节建议：

1. 左舷受风船避让右舷受风船，规则 10。
2. 同舷重叠时，上风船避让下风船，规则 11。
3. 同舷不重叠时，后船避让前船，规则 12。
4. 换舷中的船避让其他船，规则 13。
5. 有航权也要避免碰撞，规则 14。
6. 刚取得航权和改变航向时要给空间，规则 15、16。

每条规则使用同一套结构：

```text
规则名称
一句话解释
交互式局面
正确 / 错误结果
犯规提示样式
```

### 4.3 两船相遇 Demo

目标：让观众看到“如果这样撞了，谁犯规”。

预设场景：

- 左舷船和右舷船交叉。
- 下风船和上风船并行。
- 后船追前船。
- 一条船正在换舷。
- 两船接近航标。

交互方式：

- 点击“播放”，两条船自动运行。
- 点击“暂停”，停在接近碰撞点。
- 点击“判定”，显示规则解释。
- 点击“换一个结果”，切换避让成功 / 犯规碰撞。

这部分是 Demo 现场最稳定的规则展示兜底。

## 5. 项目介绍

项目介绍是“为什么做这个项目”的入口，可以理解为游戏内 PPT，但要尽量加入交互内容。

### 5.1 页面结构

建议 5 页：

1. 项目名称和一句话介绍。
2. 帆船比赛为什么难：风、水流、规则都看不见。
3. 我们的解决方案：大屏 + 蓝牙帆船手柄 + 互动模拟器。
4. 核心技术组件：船、风、水流、规则裁判。
5. 进入讲解或比赛。

### 5.2 交互式内容

项目介绍里的小动画不要另写一套逻辑，应复用底层组件：

- 背景小船复用船渲染组件。
- 风箭头复用风向标记组件。
- 水流箭头复用水流标记组件。
- 规则提示复用比赛 HUD 的规则提示组件。

这样介绍页、讲解页和比赛页看起来是同一个产品，而不是三个割裂页面。

## 6. Demo 讲解流程

Demo 讲解用于替代或承载 PPT，适合黑客松开场。

### 6.1 页面结构

Demo 讲解由 4-6 个全屏章节组成：

1. 项目是什么：专业帆船战术对抗模拟器。
2. 为什么需要：帆船比赛难点在于风、水流、航线和规则看不见。
3. 硬件亮点：特制蓝牙帆船模型手柄，1-4 通道控制 1-4 条船。
4. 战术亮点：风向摆动、风区、水流、Layline。
5. 规则亮点：自动显示规则 10-16 的犯规原因，Demo 版先做 3 秒减速。
6. 进入教学：开始学习如何操作。

### 6.2 交互方式

页面底部显示进度点和两个操作：

- 上一页。
- 下一页 / 开始教学。

大电视模式下文字要少，每页只讲一个点，右侧或背景显示动态游戏画面。

## 7. 教学模式框架

教学模式不是长教程，而是 Demo 前的快速上手。

### 7.1 教学关卡

建议 4 个短关卡：

| 关卡 | 目标 | 时长 |
| --- | --- | ---: |
| 手柄转舵 | 学会舵杆左 / 右 / 居中 | 30 秒 |
| 风向与禁航角 | 知道不能直接顶风 | 45 秒 |
| 风区与水流 | 知道环境会改变速度和航迹 | 45 秒 |
| 规则 10 | 知道左舷受风船要避让右舷受风船 | 60 秒 |

### 7.2 教学界面

教学界面由三部分组成：

- 中央：赛场和当前练习船。
- 左侧：当前目标，例如“把舵杆向右拨，让船转向右侧”。
- 右侧：即时反馈，例如“已检测到右舵 72%”。

教学完成后出现两个按钮：

- 进入比赛设置。
- 再演示一次。

## 8. 比赛设置流程

本节是第 3 节“比赛界面”的细化说明，用于后续实现具体设置页。产品入口仍然只有一个“比赛”。

比赛设置页用于选择人数、地图、难度、环境和手柄。

建议使用一步一步的向导，而不是把所有选项堆在一个页面。

```text
选择人数 -> 选择地图 -> 选择难度 -> 选择环境 -> 手柄检查 -> 开始比赛
```

## 9. 选择人数

支持 1-4 条船。

| 人数 | 用途 |
| --- | --- |
| 1 人 | 教学、计时、调试手感 |
| 2 人 | 规则演示，最容易讲清楚 |
| 3 人 | 中等复杂对抗 |
| 4 人 | 黑客松大电视主 Demo |

界面显示 4 个船位卡片：

```text
1号船  通道1  已连接
2号船  通道2  已连接
3号船  通道3  未连接 / 使用模拟输入
4号船  通道4  已连接
```

每个船位可以设置：

- 启用 / 禁用。
- 船名。
- 颜色。
- 手柄通道。
- 是否使用自动演示输入。

## 10. 选择地图

地图选择以大卡片方式展示，每张卡片显示小地图预览、预计时长和适合人数。

| 地图 | 标志数量 | 适合人数 | 说明 |
| --- | ---: | --- | --- |
| 简单场地 | 1 标 | 1-2 人 | 教学和快速试玩 |
| IO 场地 | 2 标 | 2-4 人 | 主 Demo 推荐 |
| 复杂 4 标 | 4 标 | 3-4 人 | 完整对抗 |
| 规则情景 | 预设局面 | 2 人 | 稳定演示犯规 |

地图卡片需要显示：

- 起航线。
- 航标顺序。
- 终点线。
- 主要风向。
- 水流区域。
- 风区区域。

## 11. 选择难度

难度不只是 AI 难度，而是规则严格度和辅助显示程度。

| 难度 | 适合场景 | 特点 |
| --- | --- | --- |
| 入门 | 第一次玩 / Demo 教学 | 自动调帆、显示禁航角、显示理想路线、规则只提示不频繁处罚 |
| 标准 | 黑客松主比赛 | 自动或半自动调帆、显示风流图层、规则 10 犯规减速 |
| 竞赛 | 高级玩家 | 关闭部分提示，启用更多规则判断 |

黑客松主 Demo 默认使用“标准”。

## 12. 选择环境

环境设置控制风、水流和风区。

推荐预设：

| 环境 | 风向 | 风速 | 水流 | 风区 |
| --- | --- | --- | --- | --- |
| 稳定风 | 基本不摆动 | 中等 | 无或弱 | 无 |
| 摆动风 | 左右摆动 | 中等 | 弱 | 少量 |
| 水流演示 | 稳定 | 中等 | 明显固定水流 | 少量 |
| 风区演示 | 摆动 | 中等 | 弱 | 明显强风区 |
| Demo 综合 | 摆动 | 中等偏强 | 1-2 个水流区 | 1-2 个风区 |

环境选择页要有预览动画，让观众能看到箭头变化。

## 13. 手柄检查

开始比赛前进入手柄检查页，确保现场不会因为硬件问题卡住。

### 13.1 检查内容

每个启用船位显示：

- 手柄通道。
- 连接状态。
- 当前舵杆值。
- 左舵 / 中位 / 右舵检测。
- 是否可用。

示例：

```text
1号船  通道1  已连接  右舵 64%
2号船  通道2  已连接  中位
3号船  通道3  未连接  使用模拟输入
4号船  通道4  已连接  左舵 31%
```

### 13.2 兜底策略

如果某个手柄未连接：

- 可重新扫描。
- 可换绑通道。
- 可禁用该船。
- 可使用键盘 / 自动演示输入。
- 可进入模拟控制模式，用键盘左右键或脚本控制。

手柄检查通过后，显示“开始比赛”。

## 14. 比赛准备页

点击开始比赛后，不应立刻开跑，而是进入短暂准备页。

准备页显示：

- 当前地图。
- 当前环境。
- 参赛船只。
- 规则模式。
- 起航倒计时说明。

操作：

- 开始倒计时。
- 返回设置。

黑客松 Demo 可直接自动进入 5 秒倒计时。

## 15. 比赛 HUD 框架

比赛 HUD 要为大电视优化，核心原则是：中间看比赛，四周看状态。

```text
┌──────────────────────────────────────────────┐
│ 左上：风况        顶部：比赛阶段/倒计时      右上：水流 │
│                                              │
│                                              │
│                中央：赛场                    │
│     船、航标、起航线、终点线、航迹、风区       │
│                                              │
│                                              │
│ 左侧：船队排名/规则提示              右侧：事件日志 │
│                                              │
│ 底部：1-4 号船仪表盘 + 图层/暂停/重开控制       │
└──────────────────────────────────────────────┘
```

## 16. 顶部仪表

顶部中央显示比赛阶段：

| 阶段 | 显示 |
| --- | --- |
| 准备 | `准备起航` |
| 倒计时 | `起航倒计时 00:05` |
| 比赛中 | `比赛时间 01:24` |
| 犯规慢放 | `规则判定` |
| 完赛 | `比赛结束` |

顶部可以附带当前领先船：

```text
领先：1号船 · 下一标：2标
```

## 17. 左上风况面板

显示：

- 当前全局风向箭头。
- 风向角度。
- 风速。
- 风向摆动趋势。

示例：

```text
风向 035°
风速 12 kt
趋势：右摆
```

如果当前船进入局部风区，可在船旁显示局部提示：

```text
强风区 +18%
左摆 6°
```

## 18. 右上水流面板

显示：

- 主水流方向。
- 水流速度。
- 当前地图是否启用水流。

示例：

```text
水流 ↗
0.8 kt
固定水流区
```

水流区在赛场中用半透明箭头带显示。

## 19. 船只仪表盘

底部显示 1-4 个船只仪表盘。启用几条船，就显示几个面板。

每个面板包含：

- 船号和颜色。
- 手柄通道。
- 当前名次。
- 船速。
- 舵杆比例。
- 当前受风舷：左舷受风 / 右舷受风。
- 当前航段：起航、去 1 标、去 2 标、冲线。
- 处罚状态：正常 / 减速中 2.1s。

示例：

```text
1号船  Ch1  第1名
速度 5.8 kt  舵 +42%
右舷受风  去 1 标
```

犯规减速时：

```text
2号船  Ch2  减速 2.4s
规则10：左舷未避让右舷
```

## 20. 规则提示区

规则提示必须比普通日志更醒目，建议放在左侧中部或顶部中央短暂弹出。

### 20.1 风险提示

```text
风险：2号船应避让 1号船
原因：左舷受风船接近右舷受风船
```

### 20.2 犯规提示

```text
2号船犯规
规则10：左舷受风船未避让右舷受风船
处罚：减速 3 秒
```

### 20.3 提示行为

- 风险提示用黄色。
- 犯规提示用红色或高对比描边。
- 显示 3-5 秒。
- 犯规船和被影响船在赛场中高亮。
- Demo 模式可以短暂停顿或慢放 1 秒。

## 21. 事件日志

右侧事件日志记录关键事件：

- 起航。
- 抢航。
- 绕过航标。
- 进入风区。
- 犯规。
- 减速结束。
- 完赛。

事件日志只显示最近 5-8 条，避免占满屏幕。

示例：

```text
01:12  1号船绕过 1 标
```

## 22. 图层控制

底部或右下角提供图层开关：

- 风。
- 水流。
- 航迹。
- Layline。
- 禁航角。
- 规则关系。
- 标位区。

Demo 默认打开：

- 风。
- 水流。
- 航迹。
- 规则关系。

教学时打开：

- 禁航角。
- Layline。
- 标位区。

## 23. 暂停与现场控制

现场需要几个高优先级控制：

| 控制 | 用途 |
| --- | --- |
| 暂停 / 继续 | 讲解当前局面 |
| 慢放 | 解释犯规瞬间 |
| 重开 | 快速重新演示 |
| 返回设置 | 换地图 / 换人数 |
| 触发规则场景 | 直接进入规则演示 |

这些控制可以先由鼠标 / 触屏操作，后续再映射到手柄按钮。

## 24. 赛后页面

比赛结束后进入赛后总结。

### 24.1 结果区

显示：

- 最终排名。
- 每条船总时间。
- 犯规次数。
- 被减速次数。
- 最高速度。
- 实际航程。

### 24.2 战术总结

最多显示三条：

- “1号船在右侧强风区获得速度优势。”
- “2号船违反规则10，被减速 3 秒。”
- “3号船绕标前被水流推偏，多走了一段距离。”

### 24.3 操作

- 再来一局。
- 换地图。
- 查看规则回放。
- 回到首页。

黑客松 Demo 如果时间有限，赛后页只需要排名 + 一条规则总结 + 一条战术总结。

## 26. MVP 页面优先级

P0 必须完成：

- 首页。
- 比赛入口：人数、难度 / 天气、场地、手柄检查。
- 讲解模式：船只运行逻辑、比赛规则、两船相遇 Demo。
- 项目介绍页：PPT + 交互式小船、风、水流。
- 比赛 HUD：风、水流、船只仪表盘、规则提示、事件日志。
- 赛后总结页。
- 安卓电视浏览器低性能模式：减少特效、保留核心玩法和讲解。

P1 再完成：

- 完整教学关卡。
- 规则情景选择页。
- 回放页。
- 更精细的图层设置。

P2 再完成：

- 自定义地图编辑。
- 教练创建规则场景。
- 在线多人房间。
