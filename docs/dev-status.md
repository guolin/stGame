| 5 介绍与打磨 | 5 页介绍轮播(动态模拟背景)、首页四入口、HUD 一键隐藏(H)、去外网依赖 | ✅ |

**关键约定**（改代码前必须知道）：
- 风向 `directionDeg` = 风的**来向**，0° = 屏幕正上方；比赛几何固定为风从上往下、上风标在顶部。
- 船速单位：内部 px/s，`PIXELS_PER_KNOT = 5.4` 换算节。
- 手柄 = 标准蓝牙 HID，走 **Gamepad API**（不是 Web Bluetooth）。
- 模拟必须确定性：同输入序列同结果，别在 sim 层用 `Math.random()` / `Date.now()`。

验证方式：76 个 vitest 单测（`npm test`）+ Playwright 全流程走查（首页→向导→比赛→教学三课→裁判问答→介绍轮播）。

## 2. 已知未做（按优先级）

1. **真手柄实测**：Gamepad API 通道映射逻辑未经真硬件验证，`src/game/loop/gamepadControls.ts` 可能要按实测调整。⚠️ 学生到场第一天就做。
2. **规则 18 标位区**：三倍船长圆、进区重叠锁定、碰标处罚（rules-system.md §5-6）。
3. **手柄 30 秒上手关卡**：ui-interaction-framework §7 的短教学关卡（讲解模式已有，缺"跟着做"的引导关）。
4. **比赛回放**：录 tick 快照 + 时间轴 scrub（rulesEngine 已有 2 秒航向历史可复用）。
5. **训练版转圈解脱**：一圈/两圈处罚识别（rules-system.md §10）。
6. **安卓电视低性能模式**：降采样风箭头/减航迹点的开关。
7. 体感细节：出生点贴屏幕下缘、绕标 sweep 阈值(90°)与追踪半径(170px)需真人试玩后调参。

- **D1 上午**：全员跑通环境，真手柄实测（E），每人完整玩一局。
- **D1 下午**：按 [student-guide.md](./student-guide.md) 分工进场；A 调场地环境、B/C 改内容、D 写 Pitch。
- **D2**：内容打磨 + 大电视排练一次完整 Demo 链路（介绍→讲解→2 船规则演示→4 船比赛→总结）。
- **D3（如有）**：兜底：录一条完整 Demo 视频；只修 bug 不加功能。
